Changelog

Every update to People Memory — what changed, what's new, what was fixed.

v1.1.0 Latest May 2026
Android versionCode 5 iOS build 2 Expo SDK 54 com.peoplememory.app
✦ New
  • Relationship Health Score — auto-calculated from recency, upcoming events, and profile completeness
  • Meeting Prep Cards — curated summary of what to ask and what matters before any meeting
  • Activity Journal — chronological feed of all interactions across every person
  • Voice Interaction Notes — record voice notes after meetings, timestamped per person
  • Groups — colour-coded organisation (work, family, friends, networking)
  • Contacts import — auto-fill names, numbers, and birthdays from your device address book
  • Custom dates — unlimited labelled dates per person beyond birthdays and meetings
  • Local push notifications — reminders for meetings and birthdays, fully on-device
↑ Improved
  • Avatar picker now supports device camera in addition to photo library
  • Dashboard search now matches tags, descriptions, and quick facts
  • Sort options expanded — by name, trust level, last met, and health score
  • Profile screen redesigned with tabbed sections for cleaner navigation
  • PIN auto-lock after 5 minutes of inactivity
✓ Fixed
  • Avatar not saving correctly after editing a person
  • Tutorial overlay reappearing after PIN change
  • Tag filter not resetting when navigating back to dashboard
v1.0.0 Initial Release March 2026
Android versionCode 1 iOS build 1 Expo SDK 52
✦ New
  • First public release of People Memory
  • PIN-protected vault — 4-digit PIN, SHA-256 hashed, stored only on-device
  • Add, edit, and delete people with names, tags, and avatars
  • Trust level tracking — 0 to 10 scale with red/yellow/green indicators
  • Notes — likes, dislikes, things to remember, quick facts, personal description
  • Key dates — birthday, first met, last met, next meeting
  • Avatar picker — 12 preset emoji avatars or device photo library
  • Dashboard with search, tag filter, and sort
  • One-time onboarding tutorial overlay
  • 100% offline — zero network requests, zero analytics, zero cloud
